# Avaliações

## Objetivos de aprendizagem

- Verificar compreensao conceitual de POO.
- Medir capacidade de leitura é revisão de código.
- Avaliar aplicação prática em problemas curtos.

**Tempo estimado:** 4h

## 1. Formato recomendado

- As atividades abaixo são entregues em repositórios do GitHub Classroom, não no GitHub Pages publico.
- A sequência prática segue a ordem `corrigir` -> `completar` -> `construir`.
- Cada cenário vive em um repositório starter próprio, com README, card e critérios de aceite do próprio exercicio.

| Cenário | Melhor uso | Entrega | Conceitos em foco | Observação |
|---|---|---|---|---|
| Corrigir | leitura critica e depuracao | corrigir um par C++/Python quebrado e explicar as mudanças | Git, IDE, nomes, assinaturas, leitura de erro | não depende de I/O |
| Completar | consolidar classes e contratos | terminar classes a partir de `main` pronto | encapsulamento, construtores, herança | JSON pode aparecer como contrato estatico |
| Construir | integrar camadas | fechar o dispositivo em C++ e o supervisor em Python/Streamlit | polimorfismo, integração, dashboard, colaboracao | JSON entra como troca de dados entre as duas partes |

## 2. Rubrica base

| Critério | Peso |
|---|---:|
| Correta aplicação dos conceitos de POO | 40% |
| Qualidade de modelagem | 25% |
| Clareza de código e justificativa | 20% |
| Fluxo Git e organização da entrega | 15% |

## 3. Politica de IA na avaliação

- IA pode ser usada para apoio, com rastreabilidade obrigatória.
- Cada entrega deve trazer `AI_LOG.md` ou equivalente com:
  - o que foi pedido ao agente;
  - o que foi aceito;
  - o que foi rejeitado;
  - a justificativa técnica.
- Sem defesa técnica da solução, a nota e limitada.

## 4. Critérios de leitura da entrega

- `README.md` atualizado com contexto, escopo e comandos.
- `issue` vinculada ao `pull request`.
- Branch clara por card.
- Evidência de compilação, execução ou teste.
- Histórico Git coerente com a evolução da tarefa.

## Mini-caso prático

Refatorar uma hierarquia de classes acoplada para interface + estratégia e justificar a mudança em ate 10 linhas, registrando a decisão no `AI_LOG.md`.

## Perguntas de revisão rápida

1. O que diferencia resposta correta de resposta tecnicamente defensavel?
2. Como evidenciar autoria e entendimento com uso de IA?
3. Como o histórico Git ajuda na avaliação?
4. Por que as entregas devem nascer como cards de issue?
