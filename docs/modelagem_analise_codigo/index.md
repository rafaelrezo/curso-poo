# Modelagem e Analise de Codigo

## Objetivos de aprendizagem

- Extrair classes e responsabilidades a partir de requisitos textuais.
- Identificar code smells e propor refatoracoes orientadas a objeto.
- Defender escolhas de design com argumentos tecnicos.

**Tempo estimado:** 10h (distribuidas ao longo da trilha)

## 1. Processo de modelagem

1. Identificar entidades do dominio.
2. Definir responsabilidades por classe.
3. Projetar interfaces e relacoes.
4. Validar com cenarios de uso.

## 2. Analise de codigo

- Coesao e acoplamento.
- Dependencias ciclicas.
- Classes "god object".
- Duplicacao de regras.

## 3. Refatoracao guiada

- Extrair classe.
- Extrair metodo.
- Introduzir interface.
- Aplicar composicao.

## Mini-caso pratico

Receber um codigo procedural de controle de estoque e refatorar para classes com responsabilidades claras, mantendo comportamento observavel.

## Perguntas de revisao rapida

1. Como identificar responsabilidade mal distribuida?
2. Que evidencia mostra melhora apos refatoracao?
3. Quando parar de refatorar?
